# Когда нужно выключать уведомления

> Страница обновлена автоматически по локальным ответам Wikidata. Это черновик, который удобно потом доработать вручную.

## О чём эта статья

Почему иногда полезно специально уменьшать поток сигналов и новостей.

## Что нашлось в Wikidata

- **news**. [Wikidata](https://www.wikidata.org/entity/Q17215099)
- **новости** — найдено по запросу для **news**. [Wikidata](https://www.wikidata.org/entity/Q38926)
- **ложная информация** — найдено по запросу для **misinformation**. [Wikidata](https://www.wikidata.org/entity/Q13579947)
- **бот** — найдено по запросу для **bot**. [Wikidata](https://www.wikidata.org/entity/Q55071871)

## Связи, которые видны в данных

- **news** → **не путать с** → **News**
- **новости** → **не путать с** → **Новости**
- **ложная информация** → **обрабатывается, смягчается или управляется** → **misinformation correction**
- **ложная информация** → **это частный случай понятия** → **режим отказа**

## Что это даёт для текста

По этим данным можно опереться на реальные сущности и связи вокруг темы **Когда нужно выключать уведомления**. В черновике статьи полезно объяснить, как понятия **уведомления, внимание, режим, новости** связаны друг с другом и почему это важно в повседневной цифровой жизни.

## Что ещё посмотреть в данных

- В обратных связях понятие **новости** встречается как узел с отношением **это частный случай понятия** не меньше 89 раз.
- В обратных связях понятие **новости** встречается как узел с отношением **основная тема** не меньше 20 раз.
- В обратных связях понятие **новости** встречается как узел с отношением **область деятельности** не меньше 19 раз.
